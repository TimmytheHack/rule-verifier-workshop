```text
我是广东物理类，排位32000，想学计算机，最好在广州深圳，学校稳一点，不想去太贵的中外合作。
```

## 工作簿

- Sheet：`Sheet1`
- 检测到的真实表头行：`3`
- 本 demo 必需字段均已找到：`生源地, 科类, 专业名称, 城市, 专业组最低位次1, 学费`

## Schema 边界

schema registry 只根据本 demo 需要且 Excel 中真实存在的字段构建。

缺失字段：

- `cooperation_type`：不存在。因此“中外合作”偏好在本 MVP 中不可执行。

## Deterministic Rule 验证

| 规则 | 字段 | 操作符 | 值 | 可执行 | 原因 |
|---|---|---|---|---:|---|
| `d_source_province` | `生源地` | `eq` | `广东` | True | 用户明确表达广东，且 Excel 字段 `生源地` 存在。 |
| `d_subject_type` | `科类` | `eq` | `物理` | True | 用户明确表达物理类，且 Excel 字段 `科类` 存在。 |
| `d_major_keyword` | `专业名称` | `contains` | `计算机` | True | MVP 允许精确关键词匹配。 |
| `d_city` | `城市` | `in_contains` | `['广州', '深圳']` | True | Excel 字段 `城市` 存在，且包含相关取值。 |

## Candidate Rules

| 规则 | 来源文本 | 状态 | 需要确认 | 原因 |
|---|---|---|---:|---|
| `c_safety_margin` | 学校稳一点 | pending_confirmation | True | “稳一点”是模糊偏好，必须确认安全边际后才能执行。 |
| `c_tuition_cap` | 太贵 | pending_confirmation | True | “太贵”是模糊偏好，必须确认学费上限后才能执行。 |
| `c_major_expansion` | 计算机相关扩展 | pending_confirmation | True | 超出 `计算机` 精确匹配的语义扩展必须确认。 |

## 模拟确认

- 安全边际：10%，即 `专业组最低位次1 >= 35200`。
- 学费上限：`学费 <= 20000`。
- 专业扩展：false，只匹配精确关键词 `计算机`。
- 中外合作排除：不执行，因为缺少 `cooperation_type` 字段。

## 最终查询行为

查询使用 6 条可执行规则，并采用 AND 逻辑：

1. `生源地 == 广东`
2. `科类 == 物理`
3. `专业名称 contains 计算机`
4. `城市 contains 广州 or 深圳`
5. `专业组最低位次1 >= 35200`
6. `学费 <= 20000`

查询不会从文本字段推断或过滤“中外合作”。

## 结果摘要

- 筛选结果数：`93`
- 排序方式：按 `专业组最低位次1 - 35200` 从小到大，即最接近安全线的结果优先。

## 安全检查

- 代码中不使用 LLM。
- 不进行语义专业扩展。
- 不伪造 `cooperation_type` 字段。
- Candidate rules 只有在模拟确认后才提升为可执行规则。
